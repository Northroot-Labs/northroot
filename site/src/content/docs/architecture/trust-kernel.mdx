Northroot is organized as a minimal trust kernel. The kernel does not need one
universal runtime to be useful.

## Architecture

```text
external runtime
  -> record JSON / JSONL
  -> canonical bytes
  -> hashes and event identity
  -> receipt bound to evidence
  -> optional append-only journal
  -> offline verification without a service dependency
```

## Implementation Notes

Domain-specific event types, authorization, reconciliation, execution lifecycle,
and semantic verification are external layers. They may consume the kernel, but
they do not become the kernel.

## Operational Lessons

Different runtimes can share evidence when the underlying records use stable
hashes, explicit receipts, and replayable journals.
