# ADR-007: Delta Compute Implementation Strategy

**Status:** Proposed  
**Date:** 2025-11-08 
**Deciders:** Research Synthesis (cursor + openai agents)  
**Context:** Two research reports (cursor, openai) provide complementary findings on delta/incremental compute implementation. Cursor provides comprehensive operator coverage (25 operators) with quantitative ROI (46% FinOps, 39% ETL, $1.6M total annual savings). Openai provides specific implementation hooks with exact file paths (ReuseIndexed trait, Delta Lake CDF scan, deterministic CBOR/JCS).

**Decision:** Adopt unified delta compute implementation strategy combining cursor's evidence-based domain priorities with openai's specific technical integration paths.

**Consequences:**

**Pros:**
- **Combined Evidence:** 25+ citations (cursor) + specific code paths (openai) = comprehensive foundation
- **Quantitative ROI:** $276K FinOps, $372K ETL, $684K Analytics annual savings (cursor)
- **Implementation Clarity:** Exact file paths and trait designs (openai) enable immediate development
- **Operator Coverage:** 25 operators (cursor) + 6 framework-specific operators (openai) = complete taxonomy
- **Cross-Org Verification:** Deterministic serialization (both) enables trustless compute markets

**Cons:**
- **ETL Priority Conflict:** Cursor P0 vs openai P1 → Resolved: P0 (cursor has stronger ROI evidence)
- **Operator α Values:** Some differences (e.g., partitioned sum 0.85 vs incremental sum 0.92) → Resolved: Different operators, both valid
- **Implementation Complexity:** Requires ReuseIndexed trait, α field, CBOR/JCS support, Delta Lake CDF integration

**Options Considered:**
- **Option A:** Cursor-only approach → Rejected: Missing specific implementation hooks
- **Option B:** Openai-only approach → Rejected: Missing comprehensive operator coverage and quantitative ROI
- **Option C:** Unified approach (chosen) → Combines strengths, resolves conflicts via evidence quality

**Implementation Phases:**
1. **Phase 1 (Weeks 1-2):** Core infrastructure (α field, ReuseIndexed trait, CBOR/JCS)
2. **Phase 2 (Weeks 3-4):** Framework integration (Delta Lake CDF, Spark, Dagster)
3. **Phase 3 (Weeks 5-8):** Pilot domains (FinOps, ETL, Analytics)
