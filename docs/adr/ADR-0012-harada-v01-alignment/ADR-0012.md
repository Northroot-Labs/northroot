# ADR-0012: Harada v0.1 Alignment Plan

**Date:** 2025-11-15  
**Status:** Accepted  
**Context:** Harada version 0.1 is the canonical plan for v0.1 of the engine + SDK. All
previous ADRs must be archived to avoid conflicting scope so that the organization
executes only against the Harada grid.

## Decision

- Archive ADR-0001 through ADR-0011 (status = superseded) and point them to this ADR.
- Establish ADR-0012 as the sole roadmap ADR for v0.1.
- Define six execution phases that map directly to the Harada pillars/tasks and keep
  the SDK slice extremely narrow (record_work + verify_receipt) while the engine
  continues to evolve underneath.

## Consequences

**Pros**
- **Singular authority:** No competing ADRs; Harada + ADR-0012 become the only truth
  until v0.1 completes.
- **Traceability:** Every phase references explicit Harada task IDs, keeping audits
  simple.
- **Scope discipline:** SDK surface stays slim while the core engine can continue to
  improve internally.

**Cons**
- Requires re-running ADR tooling and updates to all historical documents.
- Any future change now requires Harada + ADR updates in lockstep.
