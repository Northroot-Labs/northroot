## Phase Documentation Standards

### Phase Identification

All phase documentation must include:

1. **Phase Number**: Which phase number within the ADR/plan (e.g., "Phase 4")
2. **ADR/Plan Reference**: Link to the parent ADR or plan document
3. **Status**: Current status (Pending, In Progress, Complete, Blocked)
4. **Phase Type**: Classification per architectural alignment rules (`refactor`, `change`, `additive`)
5. **Dates**: Start date and completion date (if applicable)

### Phase Scope

- **Phases are ADR/Plan-specific**: Phase 4 of ADR-009 is different from Phase 4 of any other ADR
- **No global phase numbering**: Each ADR/plan maintains its own phase sequence
- **Cross-references**: When referencing phases, always include the ADR/plan identifier (e.g., "ADR-009 Phase 4")

### Phase Documentation Location

- Phase-specific documentation: `docs/phases/`
- Naming convention: `PHASE_<N>_<DESCRIPTION>.md` or `ADR-<N>_PHASE_<N>.md`
