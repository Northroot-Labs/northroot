# Segmented Journals and Structural Checkpoints

Status: v0.1 structural kernel contract

Segmented journals keep `.nrj` as the only journal segment format. A segmented
journal is an ordered set of `.nrj` files plus regenerable structural metadata.
It is not a projection, database, queue, workflow state, or semantic ledger.

## Layout

Recommended layout:

```text
.northroot/journals/
  segments/
    0000000000000001.nrj
    0000000000000002.nrj
    0000000000000003.nrj
```

Segment ordering is lexical by file name. The kernel does not attach domain
meaning to segment names. Rotation policy is outside the file format and should
remain operationally simple: max bytes, max events, or explicit writer request.

## Structural Manifest

A segmented journal manifest is a rebuildable summary of the ordered segments:

```json
{
  "schema": "northroot.segmented_journal_manifest.v0",
  "segment_count": 2,
  "event_count": 10000,
  "kernel_valid_event_count": 10000,
  "invalid_event_count": 0,
  "verified_prefix_event_count": 10000,
  "verified_prefix_tip_event_id": { "alg": "sha-256", "b64": "..." },
  "tip_event_id": { "alg": "sha-256", "b64": "..." },
  "prefix_digest": { "alg": "sha-256", "b64": "..." },
  "segments": []
}
```

The manifest is not authoritative by itself. It can be regenerated by scanning
the segment files and verifying their event identities.

## Structural Checkpoint

A checkpoint is a pointer to a verified prefix:

```json
{
  "schema": "northroot.journal_checkpoint.v0",
  "event_ordinal": 10000,
  "verified_prefix_event_ordinal": 10000,
  "frame_position": {
    "segment_ordinal": 2,
    "byte_offset": 1234567
  },
  "tip_event_id": { "alg": "sha-256", "b64": "..." },
  "verified_prefix_tip_event_id": { "alg": "sha-256", "b64": "..." },
  "prefix_digest": { "alg": "sha-256", "b64": "..." },
  "checkpoint_id": { "alg": "sha-256", "b64": "..." }
}
```

It says only that the prefix up to that segment/offset/ordinal was structurally
verified. It does not say any work is complete, any receipt is admissible, any
claim is true, or any projection is current.

## Stable Addressing

The v0.1 structural addressing vocabulary is:

- `EventID`: hash of canonical event JSON.
- `EventOrdinal`: position in the logical ordered event stream.
- `FramePosition`: segment ordinal plus byte offset.
- `SegmentDigest`: hash of the `.nrj` segment bytes.
- `CheckpointID`: hash of the canonical checkpoint object before the
  `checkpoint_id` field is added.

Future read models can safely record that they were derived from a checkpoint,
tip event, and event ordinal, while remaining disposable and non-authoritative.

## CLI

```bash
northroot journal verify-segments --dir .northroot/journals
northroot journal manifest --dir .northroot/journals --out journal.json
northroot journal checkpoint --dir .northroot/journals --out checkpoint.json
```

These commands verify integrity and emit structural metadata only.
